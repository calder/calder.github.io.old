Detailed Description

A simple condition implementation.

Remarks:
This class significantly simplifies the operation of normal condition variables. Most condition implementations stay fairly close to the Posix threads standard (see https://computing.llnl.gov/tutorials/pthreads/) and require three calls for each operation (this allows a few techniques that you hardy ever need). For this, AppCore provides ll*() functions that provide access to the underlying mutex and condition objects in the tradiditonal manner. These calls have serious potential for confusion, so only use them if you really know what you're doing.
If you have not used conditions before, this class should feel very natural. A condition object has either been signaled or not, and a wait() call will block until it is. signal() only clears one waiting thread while broadcast() clears them all. signal() never puts the object into a permanently "signaled" state, while broadcast() will always mark it as signaled. reset() returns the object to an unsignaled state.
Note:
Supported implementations:
  • Posix threads (THREADING_PTHREAD)
