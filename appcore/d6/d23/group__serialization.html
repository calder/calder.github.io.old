Detailed Description

This group contains all classes pertaining to AppCore’s serialization system.

Examples

Here are a few brief examples of working with the serialization system.

Example 1

SerialData objects can be created manually: 

    SerialData sd;
    sd.setType("Dog");
    sd.setName("Pooch");
    sd.setProperty<string>("Color",   "brown");
    sd.setProperty<int>   ("Weight",  40);
    sd.setProperty<bool>  ("GoodDog", true);

Example 2

Using the code from Example 1, we can call 

        printf("%s", sd.writeToString().c_str());

to get the following: 

<Dog name="Pooch">
    <Color type="string">brown</Color>
    <GoodDog type="bool">true</GoodDog>
    <Weight type="int">40</Weight>
</Dog>

The writeToString() method takes an optional argument "includeTypes", so we can call 

        printf("%s", sd.writeToString(false).c_str());

to exclude types to make it easier to read: 

<Dog name="Pooch">
    <Color>brown</Color>
    <GoodDog>true</GoodDog>
    <Weight>40</Weight>
</Dog>

Example 3

We could also save the object from Example 2 to a file: 

        sd.writeToFile("Pooch.xml");

and restore it the next time we launched the program: 

        SerialData sd;
        sd.readFromFile("Pooch.xml");

Again, you can optionally exclude types to make the output more readable. Doing so does not affect the SerialData class's ability to read the file, it just stores the data without a type until it is first referenced. This distinction is not visible to the user; you retrieve typed and untyped data in exactly the same method regardless: 

        string name = sd.getName();
        bool isGoodDog = sd.getProperty<bool>("GoodDog");
        string color = sd.getProperty<string>("Color");
        int weight = sd.getProperty<int>("Weight");
        
        string goodbad = "bad";
        if (isGoodDog) { goodbad = "good"; }
        
        printf("%s is a %s %s dog who weighs %d lbs!\n",
                name.c_str(),
                goodbad.c_str(),
                color.c_str(),
                weight
        );

which will output 

Pooch is a good brown dog who weighs 40 lbs!

Example 4

SerialData objects can also store groups of children. For example, given the following files: 

                                Pooch.xml
<Dog name="Pooch">
    <Age>4</Age>
    <Color>brown</Color>
    <Gender>m</Gender>
    <Weight>40</Weight>
</Dog>

                                Whiskers.xml
<Cat name="Whiskers">
    <Age>8</Age>
    <Color>black</Color>
    <Gender>f</Gender>
    <Weight>10</Weight>
</Dog>

You could then do the following (note how we pass the deserialization constructor true for isFile to load the two files): 

        SerialData owner;        
        owner.setType("Owner");
        owner.setName("Jim");
        owner.setProperty<int>   ("Age", 34);
        owner.setProperty<string>("Gender", "m");
        owner.setProperty<int>   ("Weight", 150);
        
        SerialData poochData    ("Pooch.xml",    true);
        SerialData whiskersData ("Whiskers.xml", true);

        owner.addChildData("Pets", poochData);
        owner.addChildData("Pets", whiskersData);

        printf("%s", owner.writeToString(false).c_str());

which outputs 

<Owner name="Jim">
    <Age>34</Age>
    <Gender>m</Gender>
    <Weight>150</Weight>
    <Pets>
        <Dog name="Pooch">
            <Age>4</Age>
            <Color>brown</Color>
            <Gender>m</Gender>
            <Weight>40</Weight>
        </Dog>
        <Cat name="Whiskers">
            <Age>8</Age>
            <Color>black</Color>
            <Gender>f</Gender>
            <Weight>10</Weight>
        </Cat>
    </Pets>
</Owner>

Note that the owner object stores copies of the two child data objects, so any changes to the originals will not affect the owner hierarchy. When a SerialData object is created as part of a hierarchy (via addChild), any setName() and moveChild() calls automatically synchronize the child's own name with the parent's name for it.
