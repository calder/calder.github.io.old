Detailed Description

Central hub for message distribution.

Note:
A MessageCenter supports three types of registration.
  • Logging registration passes all messages from all senders on to an object. As the name implies, this is primarily useful for logging features. To do this, call registerHandler() without either optional argument.
  • Global registration is where an object registers to receive all messages of a particular type from any sender. This is achieved by calling registerHandler() without the optional <messageSender> argument.
  • Local registration is where an object registers to receive only messages from a particular sender object. This is achieved by calling registerHandler() with all arguments. A single handler may only be registered at one of these scopes.
Log, global and local registrations are stored in different maps for two reasons. The first is because I was dumb and wrote it that way in the first place. The second is because I realized that it could substantially improve logging and global performance to leave it that way.

Register an object to receive messages of a particular type.

Note:
If messageSender is NULL then the handler is globally registered for all messages of that type, regardless of sender.
The handler may only be registered at the logging, global or local scopes. A combination of the three is redundant and would lead to duplicate message reception. For performance reasons, this is only checked if AppCore was compiled in Debug mode (NDEBUG off).
