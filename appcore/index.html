AppCore (Application Core Abstraction Layer) Documentation

Introduction

Welcome to the AppCore documentation. These pages contain a complete list of the various classes that AppCore provides. See the Public API for the list of user classes and subsystems. For instructions building and installing AppCore see the Installation section.

AppCore is small collection of platform independent C++ classes I have found myself writing for one thing or another over the course of my current project. While there is no attempt at cohesive coverage or even a specialized genre, many of the classes are geared towards multithreaded applications, as the STL is painfully deficient in that area. AppCore is extremely light-weight, building in just 5.884 seconds on a dual core machine. It has no external dependencies (it uses TinyXML for serialization but includes all the relevant files in its own build system).

I initially decided it would be nice to release something like this when I realized that the platform abstracted Thread, Mutex and Condition classes I had written might be of wider use. From there, AppCore has evolved into a repository of trinket classes with no higher calling than saving their authors (and hopefully you) time.

Features

All the classes in AppCore share one common goal: simplicity. Each of them was written not for some altruistic goal, but for the express purpose of saving someone time. Depending on the class, that might mean time spent coding, copy-and-pasting, debugging, dealing with platform differences, fighting with other unintuitive or overly complex libraries, or a combination of the above. Here is a brief overview of classes:

  • Messaging - AppCore provides a simple messaging system comprised of the Message, MessageCenter, and MessageHandler classes. Messages are simply strings with arbitrarily many named Arguments attached.
  • Serialization - AppCore's serialization system consists of two main classes. The SerialData class provides methods for storing primitive data and object hierarchies in dynamic containers. This allows SerialData objects to write out their complete states to an XML string or file, and load them again at will. A Serializable represents any object that stores data in a SerialData object rather than in member variables, and can thus be written to and read from files without issue. Both classes allow the user to register their own custom types to be loaded and interacted with in the same manner as built in types.
  • Threading - AppCore includes platform abstracted Thread, Mutex, and Condition classes, as well as the ThreadLoop class for simplifying the common situation of running code periodically. AppCore also includes a class ThreadSafe that is meant to simplify the task of making child classes thread-safe. There are three other classes that inherit it, TSMap, TSQueue, and TSVector that act as thread-safe wrappers for the STL's map, queue and vector classes.
  • Utilities - This includes the CountedPointer, Exception, NonCopyable, Stopwatch, and StringConverter classes, which are all fairly self-explanatory.

Installation

Grab the latest trunk from AppCore's Bazaar repository (https://code.launchpad.net/~judgmentday/appcore/trunk) using: 

bzr checkout --lightweight lp:appcore

Open a terminal window and cd to the 'appcore' directory. Create and cd to a 'build' subdirectory, and build and install the library: 

cmake ..
make
sudo make install

Contributing

By definition, AppCore has no grand development plan or direction. If I write something I think is useful, I will add it. If someone else has something they think is useful, I would greatly appreciate the contribution. The only requirements are that the code be clean, well-commented, easy to use, and not take itself too seriously. Speaking of which, it must also be subject to AppCore's licensing agreement.
