The Sorting Hat

Assignment goals:
  1. Introduce object orientation and efficiency.
  2. Teach some Java basics: types, functions and loops.
  3. Have fun with sorting!
With that in mind, the assignment intentionally avoids many "slick" tricks introduced in future assignments. Everything except the array and GUI code lives in a single file composed of static functions. This gives an "outside-in" introduction to object orientation, where students get practice using an Array class before writing their own.

The Even Cooler Sorting Hat

Assignment goals:
  1. Introduce polymorphism and recursion.
  2. Analyze running times of recursive functions.
  3. Play around with merge sort and quick sort!
This assignment is designed to be as slick as possible to hopefully inspire some appreciation for the benefits of polymorphism. Sorting algorithms and displays have both been abstracted away into interfaces so students can compare any set of algorithms using any display. And while each sorting algorithm is its own class, they have no instance variables or state yet.

Quackers

Assignment goals:
  1. Introduce interfaces and unit testing.
  2. Write a data structure.
  3. Mess around with arrays and lists and queues and stacks oh my!
Students will implement a simple double-ended queue interface. The assignment uses no visualization because of the open-ended implementation, but instead includes a bunch of unit tests to help with debugging. Once they can pass all tests, there is a speed trial that will automatically submit times to a high score server for a class wide competition.
